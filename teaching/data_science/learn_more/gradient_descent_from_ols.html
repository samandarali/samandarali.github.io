Gradient Descent from Traditional Regression

Main idea: In ordinary least squares, we minimize the sum of squared residuals by solving the first-order conditions exactly. In gradient descent, we use the same objective function and the same derivatives, but instead of solving the derivative equation exactly, we move step by step until the derivative becomes close to zero.

1. Traditional Regression: What You Already Know

Suppose we have the simple linear regression model:

\[ \hat y_i = \beta_0 + \beta_1 x_i \]

The residual is:

\[ e_i = y_i - \hat y_i = y_i - (\beta_0 + \beta_1x_i) \]

The sum of squared errors is:

\[ S(\beta_0, \beta_1)=\sum_{i=1}^{n}(y_i-\beta_0-\beta_1x_i)^2 \]

In the traditional OLS approach, we take derivatives and set them equal to zero:

\[ \frac{\partial S}{\partial \beta_0}=0, \qquad \frac{\partial S}{\partial \beta_1}=0 \]

Then we solve these equations exactly and obtain the OLS coefficients.

Important: OLS and gradient descent start from the same objective function. The difference is not the goal. The difference is the method used to reach the minimum.

2. Why Use Gradient Descent?

For small ordinary least squares problems, the closed-form OLS solution is usually fine. Gradient descent becomes useful when:

  • the dataset is too large for matrix inversion,
  • there are many features,
  • the matrix \(X'X\) is not invertible,
  • the model has no closed-form solution, such as logistic regression or neural networks.

The closed-form OLS solution is:

\[ \hat\beta = (X'X)^{-1}X'y \]

This requires matrix inversion. Gradient descent avoids matrix inversion.

3. The Key Confusion: What Do We Differentiate?

The derivative is not taken after you compute one number like \(S=850\). The derivative is taken from the function \(S(\beta_0,\beta_1)\) before plugging in specific values for \(\beta_0\) and \(\beta_1\).

The regression model is:

\[ \hat y_i = \beta_0 + \beta_1x_i \]

So the residual is:

\[ e_i = y_i - (\beta_0 + \beta_1x_i) \]

This residual is still a function of \(\beta_0\) and \(\beta_1\). It is not a fixed number until you choose coefficient values.

Write SSE as a Function

\[ S(\beta_0,\beta_1)=\sum_{i=1}^{n}(y_i-\beta_0-\beta_1x_i)^2 \]

Using the three actual observations:

\[ S(\beta_0,\beta_1) =(2-\beta_0-\beta_1)^2 +(3-\beta_0-2\beta_1)^2 +(5-\beta_0-3\beta_1)^2 \]

This is the formula whose derivative we calculate.

4. Derivative with Respect to \(\beta_1\)

Now differentiate the function with respect to \(\beta_1\):

\[ \frac{\partial S}{\partial \beta_1} = -2(2-\beta_0-\beta_1) -4(3-\beta_0-2\beta_1) -6(5-\beta_0-3\beta_1) \]

This derivative is still a function. It depends on \(\beta_0\) and \(\beta_1\).

Now Substitute the Current Guess

Suppose gradient descent starts with:

\[ \beta_0=0, \qquad \beta_1=0 \]

Substitute them into the derivative:

\[ \frac{\partial S}{\partial \beta_1} = -2(2)-4(3)-6(5)=-46 \]
At the point \((0,0)\), the slope with respect to \(\beta_1\) is \(-46\). This means increasing \(\beta_1\) will reduce the SSE.

5. Gradient Descent Update Rule

The update rule is:

\[ \beta_j \leftarrow \beta_j - \alpha \frac{\partial S}{\partial \beta_j} \]

where:

  • \(\beta_j\) is the coefficient being updated,
  • \(\alpha\) is the learning rate,
  • \(\frac{\partial S}{\partial \beta_j}\) is the derivative of the loss with respect to that coefficient.

Using \(\alpha=0.01\):

\[ \beta_1 = 0 - 0.01(-46)=0.46 \]

Then we evaluate the same derivative formula again, but now using the updated coefficient value. The derivative formula does not change; only the values plugged into it change.

7. Correct Understanding of the Derivative

You derive the derivative formula once. Then, at each iteration, you substitute the current values of the coefficients into that formula to compute the current gradient. You update the coefficients and repeat until the gradient is close to zero or the cost function stops improving meaningfully.

In Python-like logic:

initialize beta0 = 0
initialize beta1 = 0
choose learning_rate = 0.01

repeat many times:
    compute predictions
    compute residuals
    compute derivative with respect to beta0
    compute derivative with respect to beta1
    update beta0
    update beta1

stop when the gradient is close to zero
or when SSE changes very little

8. What Is the Learning Rate?

The learning rate \(\alpha\) controls the size of each step.

\[ \beta_j \leftarrow \beta_j - \alpha \frac{\partial S}{\partial \beta_j} \]

The gradient tells us the direction. The learning rate tells us how far to move in that direction.

9. How Is the Learning Rate Chosen?

There is no universal formula for the best learning rate. It is a hyperparameter chosen by the analyst.

In practice, you test different values and monitor whether the loss decreases smoothly.

Method 1: Try Several Values

Train the model with different learning rates and compare the loss curve.

Learning RatePossible Result
Too smallVery slow convergence
ReasonableSmooth decrease in loss
Too largeOscillation or divergence

Method 2: Learning Rate Schedule

Start with a larger learning rate and reduce it during training.

Start: 0.1
Later: 0.05
Later: 0.01
Later: 0.001

This gives larger steps at the beginning and smaller steps near the minimum.

Method 3: Adaptive Optimizers

Modern machine learning often uses optimizers that adjust the effective learning rate automatically. Examples include:

  • SGD with Momentum
  • RMSProp
  • Adam
  • AdamW

10. Final Intuition

Think of gradient descent as walking downhill in fog. The gradient tells you which direction is downhill. The learning rate tells you how large your step should be. You continue walking until the slope is nearly flat.

OLS solves the problem analytically. Gradient descent solves the problem numerically. Both are trying to minimize the same loss function.
